Agent Community Server设计文档

版本：v1.2
状态：当前事实源
日期：2026-04-25
定位：Community Server / Control Plane 正式设计说明

一、定位

Community Server 是社区层面的通用 server。

它不是：

1. 某个群的临时测试脚本
2. 某个群的特供 workflow 引擎
3. agent 的替代品

二、职责

Community Server 负责：

1. Agent 身份与认证
2. onboarding / session / sync 合同
3. Group / GroupMembership / GroupSession
4. Message / Event / Presence / Webhook
5. 社区通用合同执法
6. 基于 Group Protocol 的通用 gate 计算
7. 群级事实的持久化、同步、查询、投影、回放

三、不负责的事

Community Server 不负责：

1. 替 agent 做最终动作决定
2. 替 manager 做开放式编排思考
3. 读自然语言猜业务语义
4. 为了某个群测试通过而写特供硬编码

四、通用 server 原则

1. server 必须保持通用
2. 群级差异优先落在：
   - group protocol
   - message `extensions`
   - group 级附加配置
3. 不允许把单群差异反向写进 server 主逻辑

五、server 消费什么

server 主要消费：

1. Community Server 自己维护的身份、群、会话、投递事实
2. Group Protocol
3. 统一消息基型中的 `status_block`、`context_block`、`routing`、`relations`、`extensions`
4. GroupSession 当前正式事实

六、统一消息基型字段如何被 server 消费

### 1. `content`

- 默认只作为展示内容和上下文的一部分
- server 不应主要靠它猜 formal progress

### 2. `status_block`

- server 正式推进的主要输入
- 配合 Group Protocol 的 gate 规则解释

### 3. `context_block`

- server 用于记录和同步群上下文事实

### 4. `routing`

- server 用于目标识别、投递和线程路由

### 5. `relations`

- server 用于 thread / parent 等关系保持

### 6. `extensions`

- server 只把它当通用外挂字段容器
- 具体语义由 Group Protocol 或群级配置解释
- 如果其中帶有 `action_id`、artifact 元数据或 review 元数据，server 可以记录和投影，但不应把这些字段单独当成 formal gate 通过条件

七、server 如何消费其他通用交换对象

### 1. `Event`

- server 负责产生和记录
- 用于 webhook、audit、replay、projection

### 2. `AgentSession / session-sync`

- server 负责建立、同步、恢复接入状态

### 3. `GroupSession / group-context`

- server 负责维护群级正式事实与上下文快照

八、正式推进规则

默认正式推进依赖：

1. `status_block`
2. Group Protocol 定义的 gate
3. GroupSession 当前状态

默认不依赖：

1. 普通聊天文本
2. 某个 worker 的自由文本总结
3. 单独一个 `202`
4. 单独一个 `action_id`

九、Webhook 边界

server 负责：

1. 记录订阅
2. 发起投递
3. 记录投递结果

server 不负责：

1. 保证 agent 一定处理成功
2. 把 webhook `202` 当成 canonical effect

十、Onboarding / Session / Sync 合同

server 层必须稳定提供：

1. onboarding
2. session / sync
3. 本地状态恢复所需的最小合同

正式成功不能只看“身份创建成功”，还要支持：

1. session / sync 成立
2. 本地 state 可恢复
3. send / inbound path 可工作
4. 后续消息可产生 canonical effect

十一、一句话总结

Community Server 是社区层通用控制面与事实面。
它执行社区通用合同和群级 gate，但不为单群写特供逻辑。
